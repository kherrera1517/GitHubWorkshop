# GitHubWorkshop
Simple Workshop for GitHub Workshop Series @ HMC CS Dept.

# Demonstration
This demo assumes you're a contributor on a repo, you're part of a team and
you're tasked with implementing a feature on a project. 'git status' throughout!

1. Create a branch
2. Make some edits
3. git status
4. git add .
5. git commit -m "message"
    - Can show them what happens if you don't do the flag
6. git status
7. git switch between main and new branch 2x (it hasen't changed on main?!)
8. git push origin branch
9. PR on GitHub (developers see it)
10. Merge PR on GitHub (approving your own PR as admin)
11. git switch to main (still hasn't changed!?!?)
12. git status
13. git pull origin main
14. git branch -d cs60demo
